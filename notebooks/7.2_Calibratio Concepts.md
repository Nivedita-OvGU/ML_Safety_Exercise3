# Exercise 7.2: Calibration and ECE

* A classifier is well-calibrated when the model’s confidence matches its empirical accuracy (actual accuracy observed on real data)

* Definition:

  * Calibration means alignment of the model’s confidence with its empirical accuracy**.
  * A model is calibrated if, among all predictions made with confidence `p`, a fraction `p` are actually correct.

* Example:

  * If a weather forecaster says “70% rain” on many days:

    * If it rains on 70% of those days, the forecaster is well-calibrated.
    * If it rains on only 40% of those days, the forecaster is overconfident.
    * If it rains on 90% of those days, the forecaster is underconfident.

* Expected Calibration Error, ECE, measures the average calibration gap.

* How ECE is computed:

  * Bin predictions by confidence.
  * For each bin, compare:

    * the average confidence in that bin
    * the empirical accuracy in that bin
  * Compute the absolute gap between accuracy and confidence.
  * Weight this gap by the number of samples in the bin.
  * Sum the weighted gaps over all bins.

* Formula:

  `ECE = Σ (|Bm| / n) × |acc(Bm) - conf(Bm)|`

* Here:

  * `M` = number of bins
  * `Bm` = predictions in bin `m`
  * `n` = total number of datapoints
  * `acc(Bm)` = accuracy in bin `m`
  * `conf(Bm)` = average confidence in bin `m`

* Small ECE means better calibration.

* Large ECE means the model confidence is not trustworthy.

* The lecture also notes that ECE is one aggregate number, so it can hide failures in a subgroup or confidence range.

# Exercise 7.3: Cost-Optimal Downstream Decisions

* Given:

  * `p = p(pedestrian present | x)`
  * `1 - p = p(no pedestrian | x)`
  * False positive cost: `CFP = 1`
  * False negative cost: `CFN = 100`

# 1. Expected loss of each action

* If the autopilot chooses brake:

  * Cost is `0` when pedestrian is present.
  * Cost is `CFP = 1` when no pedestrian is present.
  * So:

E[L | brake] = p · 0 + (1 - p) · 1
E[L | brake] = 1 - p

* If the autopilot chooses continue:

  * Cost is `CFN = 100` when pedestrian is present.
  * Cost is `0` when no pedestrian is present.
  * So:

E[L | continue] = p · 100 + (1 - p) · 0
E[L | continue] = 100p

# 2. Threshold where both losses are equal

* Set both expected losses equal: 1 - p = 100p
* Solve:

1 = 101p
p = 1 / 101
so,
τ* = 1 / 101
-- Compute τ*

* Final threshold: τ* = 0.0099

* This means the autopilot should already brake when pedestrian probability is about 1%.

* Compared to standard argmax:

standard threshold = 0.5
cost-optimal threshold = 0.0099

* So the cost-optimal threshold is much lower because missing a pedestrian is much more important than braking unnecessarily.

# 4. Why calibration matters

* `τ*` only gives the optimal decision if `p` is well-calibrated.
* Well-calibrated means the predicted probability really matches the true empirical probability.
* If the model says `p = 0.01`, then about 1% of such cases should actually contain pedestrians.
* If the model is overconfident or miscalibrated, the probability is not trustworthy.
* Then the autopilot may continue when it should brake, or brake too often when it is not needed.
* For an overconfident model, a low predicted pedestrian probability may look safe, but it can still be wrong.
